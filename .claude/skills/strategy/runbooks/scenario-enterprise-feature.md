# 操作手册：企业级功能开发

> **模式**：NEXUS-Sprint | **时长**：6-12 周 | **智能体**：20-30

---

## 场景

你要在一个现有的企业级产品上加一个大功能。合规、安全、质量门禁没有商量余地。多方利益相关方需要对齐。功能必须和现有系统无缝集成。

## 执行计划

### 第 1 阶段：需求与架构（第 1-2 周）

```
第 1 周：利益相关方对齐
├── 项目牧羊人 → 利益相关方分析 + 沟通计划
├── UX 研究员 → 功能需求的用户调研
├── 法务合规员 → 合规要求扫描
├── 高级项目经理 → 需求转任务
└── 财务追踪员 → 预算框架

第 2 周：技术架构
├── UX 架构师 → UX 基础 + 组件架构
├── 后端架构师 → 系统架构 + 集成方案
├── UI 设计师 → 组件设计 + 设计系统更新
├── Sprint 排序师 → RICE 评分的待办列表
├── 品牌守护者 → 品牌影响评估
└── 质量门禁：架构评审（项目牧羊人 + 现实检验者）
```

### 第 2 阶段：基础搭建（第 3 周）

```
├── DevOps 自动化师 → 功能分支流水线 + 功能开关
├── 前端开发者 → 组件脚手架
├── 后端架构师 → API 骨架 + 数据库迁移
├── 基础设施运维师 → 预发布环境搭建
└── 质量门禁：基础验证（证据收集者）
```

### 第 3 阶段：构建（第 4-9 周）

```
Sprint 1-3（第 4-9 周）：
├── 智能体编排者 → 开发-测试循环管理
├── 前端开发者 → UI 实现（逐任务）
├── 后端架构师 → API 实现（逐任务）
├── 高级开发者 → 复杂/高级功能
├── 证据收集者 → 每个任务都 QA（截图）
├── API 测试员 → 每个 API 任务都验证端点
├── 实验追踪员 → 关键功能搭 A/B 测试
│
├── 双周：
│   ├── 项目牧羊人 → 利益相关方状态更新
│   ├── 高管摘要生成器 → 高管简报
│   └── 财务追踪员 → 预算追踪
│
└── Sprint 回顾 + 利益相关方演示
```

### 第 4 阶段：加固（第 10-11 周）

```
第 10 周：证据收集
├── 证据收集者 → 全量截图套件
├── API 测试员 → 完整回归套件
├── 性能基准师 → 10 倍流量压力测试
├── 法务合规员 → 最终合规审计
├── 测试结果分析师 → 质量指标仪表盘
└── 基础设施运维师 → 生产就绪检查

第 11 周：最终判定
├── 现实检验者 → 集成测试（默认：需要改进）
├── 需要的话修一修（2-3 天）
├── 重新验证
└── 高管摘要生成器 → 上线/不上线建议
```

### 第 5 阶段：灰度上线（第 12 周）

```
├── DevOps 自动化师 → 金丝雀部署（5% → 25% → 100%）
├── 基础设施运维师 → 实时监控
├── 数据分析师 → 功能采用率追踪
├── 客服响应者 → 新功能用户支持
├── 反馈分析师 → 早期反馈收集
└── 高管摘要生成器 → 上线报告
```

## 利益相关方沟通节奏

| 受众 | 频率 | 智能体 | 格式 |
|------|------|--------|------|
| 高管赞助人 | 双周 | 高管摘要生成器 | SCQA 摘要（不超过 500 字） |
| 产品团队 | 每周 | 项目牧羊人 | 状态报告 |
| 工程团队 | 每日 | 智能体编排者 | 流水线状态 |
| 合规团队 | 每月 | 法务合规员 | 合规状态 |
| 财务 | 每月 | 财务追踪员 | 预算报告 |

## 质量要求

| 要求 | 阈值 | 验证方 |
|------|------|--------|
| 代码覆盖率 | > 80% | 测试结果分析师 |
| API 响应时间 | P95 < 200ms | 性能基准师 |
| 无障碍 | WCAG 2.1 AA | 证据收集者 |
| 安全 | 零严重漏洞 | 法务合规员 |
| 品牌一致性 | 95%+ 合规 | 品牌守护者 |
| 需求合规 | 100% | 现实检验者 |
| 承压能力 | 当前流量的 10 倍 | 性能基准师 |

## 风险管理

| 风险 | 概率 | 影响 | 缓解措施 | 负责人 |
|------|------|------|---------|--------|
| 集成复杂度 | 高 | 高 | 尽早做集成测试，每个 Sprint 都跑 API 测试员 | 后端架构师 |
| 范围蔓延 | 中 | 高 | Sprint 排序师用 MoSCoW 管控，项目牧羊人管理变更 | Sprint 排序师 |
| 合规问题 | 中 | 紧急 | 法务合规员从第 1 天就参与 | 法务合规员 |
| 性能退化 | 中 | 高 | 每个 Sprint 都跑性能基准师测试 | 性能基准师 |
| 利益相关方不对齐 | 低 | 高 | 双周高管简报，项目牧羊人协调 | 项目牧羊人 |
