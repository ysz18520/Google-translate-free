# 第 1 阶段手册 — 策略与架构

> **时长**：5-10 天 | **智能体**：8 个 | **守门人**：工作室制片人 + 现实检验者

---

## 目标

定义我们要做什么、怎么搭、成功标准是什么——一行代码都不写之前搞定这些。每个架构决策都有记录，每个功能都排过优先级，每一笔钱都算清楚。

## 前提条件

- [ ] 第 0 阶段质量门禁通过（"继续"决策）
- [ ] 收到第 0 阶段交接包
- [ ] 利益相关方在项目范围上已对齐

## 智能体激活顺序

### 第一步：策略定调（第 1-3 天，并行）

#### 工作室制片人 — 策略组合对齐
```
激活工作室制片人，做 [项目] 的策略组合对齐。

输入：第 0 阶段高管摘要 + 市场分析报告
交付要求：
1. 策略组合计划及项目定位
2. 愿景、目标和 ROI 指标
3. 资源配置策略
4. 风险收益评估
5. 成功标准和里程碑定义

对齐目标：组织层面的策略目标
格式：策略组合计划模板
时间线：3 天
```

#### 品牌守护者 — 品牌识别系统
```
激活品牌守护者，做 [项目] 的品牌识别建设。

输入：第 0 阶段 UX 调研（画像、旅程图）
交付要求：
1. 品牌基础（使命、愿景、价值观、品牌个性）
2. 视觉识别系统（颜色、字体、间距用 CSS 变量）
3. 品牌语调和信息架构
4. Logo 系统规范（如果是新品牌）
5. 品牌使用规范

格式：品牌识别系统文档
时间线：3 天
```

#### 财务追踪员 — 预算和资源规划
```
激活财务追踪员，做 [项目] 的财务规划。

输入：工作室制片人策略计划 + 第 0 阶段技术栈评估
交付要求：
1. 按类别分的完整项目预算
2. 资源成本预测（智能体、基础设施、工具）
3. ROI 模型及盈亏平衡分析
4. 现金流时间线
5. 财务风险评估及应急储备

格式：财务计划 + ROI 预测
时间线：2 天
```

### 第二步：技术架构（第 3-7 天，并行，第一步产出就绪后）

#### UX 架构师 — 技术架构 + UX 基础
```
激活 UX 架构师，做 [项目] 的技术架构。

输入：品牌守护者视觉识别 + 第 0 阶段 UX 调研
交付要求：
1. CSS 设计系统（变量、token、比例尺）
2. 布局框架（Grid/Flexbox 模式、响应式断点）
3. 组件架构（命名规范、层级结构）
4. 信息架构（页面流程、内容层级）
5. 主题系统（亮色/暗色/跟随系统切换）
6. 无障碍基础（WCAG 2.1 AA 基线）

要创建的文件：
- css/design-system.css
- css/layout.css
- css/components.css
- docs/ux-architecture.md

格式：开发者可直接使用的基础包
时间线：4 天
```

#### 后端架构师 — 系统架构
```
激活后端架构师，做 [项目] 的系统架构。

输入：第 0 阶段技术栈评估 + 合规要求
交付要求：
1. 系统架构文档
   - 架构模式（微服务/单体/Serverless/混合）
   - 通信模式（REST/GraphQL/gRPC/事件驱动）
   - 数据模式（CQRS/Event Sourcing/CRUD）
2. 数据库 Schema 设计及索引策略
3. API 设计文档及版本策略
4. 认证和授权架构
5. 安全架构（纵深防御）
6. 扩展方案（水平扩展策略）

格式：系统架构文档
时间线：4 天
```

#### AI 工程师 — ML 架构（如果需要）
```
激活 AI 工程师，做 [项目] 的 ML 系统架构。

输入：后端架构师系统架构 + 第 0 阶段数据审计
交付要求：
1. ML 系统设计
   - 模型选型和训练策略
   - 数据管道架构
   - 推理策略（实时/批处理/边缘）
2. AI 伦理和安全框架
3. 模型监控和重训练计划
4. 与主应用的集成点
5. ML 基础设施成本预测

前提：只在项目包含 AI/ML 功能时激活
格式：ML 系统设计文档
时间线：3 天
```

#### 高级项目经理 — 需求到任务的转化
```
激活高级项目经理，为 [项目] 创建任务清单。

输入：全部第 0 阶段文档 + 架构文档（已有的部分）
交付要求：
1. 完整任务清单
   - 引用需求文档的原文（不加额外功能）
   - 每个任务有明确的验收标准
   - 标注任务间的依赖
   - 工作量估算（故事点或小时）
2. 工作分解结构
3. 关键路径识别
4. 实现阶段的风险登记表

规则：
- 不要加需求文档里没有的功能
- 引用需求的原文
- 工作量估算要实事求是

格式：带验收标准的任务清单
时间线：3 天
```

### 第三步：排优先级（第 7-10 天，串行，第二步完成后）

#### Sprint 排序师 — 功能排序
```
激活 Sprint 排序师，做 [项目] 的待办排序。

输入：
- 高级项目经理 → 任务清单
- 后端架构师 → 系统架构
- UX 架构师 → UX 架构
- 财务追踪员 → 预算框架
- 工作室制片人 → 策略计划

交付要求：
1. RICE 评分的待办列表（Reach, Impact, Confidence, Effort）
2. 按速度估算的 Sprint 分配
3. 依赖图 + 关键路径
4. MoSCoW 分类（Must/Should/Could/Won't）
5. 发布计划和里程碑映射

验证：工作室制片人确认策略对齐
格式：排好序的 Sprint 计划
时间线：2 天
```

## 质量门禁检查清单

| # | 标准 | 证据来源 | 状态 |
|---|------|---------|------|
| 1 | 架构覆盖 100% 的需求 | 高级项目经理任务清单与架构交叉核对 | |
| 2 | 品牌系统完整（logo、颜色、字体、语调） | 品牌守护者交付物 | |
| 3 | 所有技术组件都有实现路径 | 后端架构师 + UX 架构师文档 | |
| 4 | 预算已批准且在约束范围内 | 财务追踪员计划 | |
| 5 | Sprint 计划基于速度估算、切合实际 | Sprint 排序师待办列表 | |
| 6 | 安全架构已定义 | 后端架构师安全文档 | |
| 7 | 合规要求已融入架构 | 法律要求与技术决策的映射 | |

## 门禁决策

**需要双签**：工作室制片人（策略层面）+ 现实检验者（技术层面）

- **通过**：带着完整架构包进入第 2 阶段
- **修改**：具体事项需要返工（回到对应步骤）
- **重来**：架构有根本性问题（重新走第 1 阶段）

## 交接给第 2 阶段

```markdown
## 第 1 阶段 → 第 2 阶段交接包

### 架构包：
1. 策略组合计划（工作室制片人）
2. 品牌识别系统（品牌守护者）
3. 财务计划（财务追踪员）
4. CSS 设计系统 + UX 架构（UX 架构师）
5. 系统架构文档（后端架构师）
6. ML 系统设计（AI 工程师——如果有的话）
7. 完整任务清单（高级项目经理）
8. 排好序的 Sprint 计划（Sprint 排序师）

### 给 DevOps 自动化师的：
- 后端架构师的部署架构
- 系统架构中的环境要求
- 基础设施的监控要求

### 给前端开发者的：
- UX 架构师的 CSS 设计系统
- 品牌守护者的品牌识别
- UX 架构师的组件架构
- 后端架构师的 API 文档

### 给后端架构师（继续工作）的：
- 数据库 Schema 准备好部署
- API 骨架准备好实现
- 认证系统架构已定义
```

---

*当工作室制片人和现实检验者都在架构包上签字后，第 1 阶段结束。*
