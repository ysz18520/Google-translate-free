# 第 5 阶段手册 — 上线与增长

> **时长**：2-4 周（T-7 到 T+14）| **智能体**：12 个 | **守门人**：工作室制片人 + 数据分析师

---

## 目标

协调所有渠道同时执行上市动作。上线时打出最大声量。每个营销智能体协同开火，工程团队保障稳定性。

## 前提条件

- [ ] 第 4 阶段质量门禁通过（现实检验者判定"就绪"）
- [ ] 收到第 4 阶段交接包
- [ ] 生产部署方案已批准
- [ ] 营销内容管道就绪（来自第 3 阶段轨道 B）

## 上线时间线

### T-7：上线准备周

#### 内容与活动准备（并行）

```
激活内容创作者：
- 定稿所有上线内容（博客、着陆页、邮件序列）
- 在发布平台排好队列
- 准备好预期问题的回复模板
- 制定上线当天的实时内容计划

激活社交媒体策略师：
- 定稿跨平台活动素材
- 排好预热内容的发布时间
- 协调 KOL 合作
- 准备各平台定制版内容

激活增长黑客：
- 武装病毒式传播机制（推荐码、分享激励）
- 配置增长实验追踪
- 搭好漏斗分析
- 准备获客渠道预算

激活应用商店优化师（如果是移动端）：
- 定稿商店页面（标题、描述、关键词、截图）
- 提交应用审核（如适用）
- 准备上线当天的 ASO 调整
- 配置应用内评价提示
```

#### 技术准备（并行）

```
激活 DevOps 自动化师：
- 准备蓝绿部署
- 验证回滚流程
- 配置功能开关做灰度发布
- 端到端测试部署流水线

激活基础设施运维师：
- 配置 10 倍预期流量的自动伸缩
- 验证监控和告警阈值
- 测试灾难恢复流程
- 准备事故响应手册

激活项目牧羊人：
- 给所有智能体分发上线检查清单
- 确认所有依赖已解决
- 搭建上线当天通讯频道
- 向利益相关方通报上线计划
```

### T-0：上线当天

#### 第 0 小时：部署

```
激活 DevOps 自动化师：
1. 执行蓝绿部署到生产环境
2. 对所有服务跑健康检查
3. 验证数据库迁移完成
4. 确认所有端点有响应
5. 把流量切到新部署
6. 监控错误率 15 分钟
7. 确认：部署成功 或 回滚

激活基础设施运维师：
1. 实时监控所有系统指标
2. 观察流量峰值和伸缩事件
3. 追踪错误率和响应时间
4. 任何阈值被突破时告警
5. 确认：系统稳定
```

## 门禁决策

**双签**：工作室制片人（策略层面）+ 数据分析师（数据层面）

- **稳定**：产品已上线，系统稳定，增长渠道活跃 → 激活第 6 阶段
- **紧急**：需要工程紧急响应的大问题 → 热修复循环
- **回滚**：根本性问题 → 撤回部署，回到第 4 阶段
