# 第 6 阶段手册 — 运营与演进

> **时长**：持续进行 | **智能体**：12+（轮换）| **治理**：工作室制片人

---

## 目标

持续运营 + 持续改进。产品已经上线了——现在让它活得好。这个阶段没有结束日期，只要产品还在市场上就一直跑。

- [ ] 第 5 阶段质量门禁通过（稳定上线）
- [ ] 收到第 5 阶段交接包
- [ ] 运营节奏已建立
- [ ] 基线指标已记录

## 持续改进循环

```
测量（数据分析师）
    │
    ▼
分析（反馈分析师 + 数据分析师）
    │
    ▼
规划（Sprint 排序师 + 工作室制片人）
    │
    ▼
构建（第 3 阶段开发-测试循环 — 小周期）
    │
    ▼
验证（证据收集者 + 现实检验者）
    │
    ▼
部署（DevOps 自动化师）
    │
    ▼
测量（回到起点）
```

### 第 6 阶段的功能开发

新功能走一个压缩版的 NEXUS 周期：

```
1. Sprint 排序师从待办列表选功能
2. 合适的开发智能体实现
3. 证据收集者验证（开发-测试循环）
4. DevOps 自动化师部署（功能开关或直接发布）
5. 实验追踪员监控（适用时做 A/B 测试）
6. 数据分析师衡量效果
7. 反馈分析师收集用户反应
```

## 事故响应协议

### 严重度分级

| 级别 | 定义 | 响应时间 | 决策权 |
|------|------|---------|--------|
| **P0 — 紧急** | 服务挂了、数据丢了、安全事件 | 立即 | 工作室制片人 |
| **P1 — 高** | 主要功能挂了、性能严重下降 | < 1 小时 | 项目牧羊人 |
| **P2 — 中** | 次要功能有问题、有临时方案 | < 4 小时 | 智能体编排者 |
| **P3 — 低** | 样式问题、小不便 | 下个 Sprint | Sprint 排序师 |

### 事故响应流程

```
发现（基础设施运维师 或 客服响应者）
    │
    ▼
分诊（智能体编排者）
    ├── 确定严重度（P0-P3）
    ├── 指派响应团队
    └── 通知利益相关方
    │
    ▼
响应
    ├── P0：基础设施运维师 + DevOps 自动化师 + 后端架构师
    ├── P1：相关开发智能体 + DevOps 自动化师
    ├── P2：相关开发智能体
    └── P3：加到 Sprint 待办列表
    │
    ▼
解决
    ├── 修复已实现并部署
    ├── 证据收集者验证修复
    └── 基础设施运维师确认系统稳定
    │
    ▼
事后复盘
    ├── 工作流优化师主持回顾
    ├── 根因分析已记录
    ├── 预防措施已确定
    └── 流程改进已落实
```

## 增长运营

### 月度增长复盘（增长黑客主导）

```
1. 渠道表现分析
   - 按渠道的获客（自然、付费、推荐、社交）
   - 按渠道的 CAC
   - 按漏斗阶段的转化率
   - LTV:CAC 比率趋势

2. 实验结果
   - 已完成的 A/B 测试及结论
   - 统计显著性验证
   - 胜出方案的落地状态
   - 新实验管道

3. 留存分析
   - 队列留存曲线
   - 流失风险识别
   - 召回活动效果
   - 功能采用指标

4. 增长路线图更新
   - 下月增长实验
   - 渠道预算重新分配
   - 新渠道探索
   - 病毒系数优化
```

## 财务运营

### 月度财务复盘（财务追踪员）

```
1. 营收分析
   - MRR/ARR 追踪
   - 按细分/套餐的营收
   - 扩展营收
   - 流失对营收的影响

2. 成本分析
   - 基础设施成本
   - 按渠道的营销支出
   - 团队/资源成本
   - 工具和服务成本

3. 单位经济
   - CAC 趋势
   - LTV 趋势
   - LTV:CAC 比率
   - 回本周期

4. 预测
   - 营收预测（滚动 3 个月）
   - 成本预测
   - 现金流预测
   - 预算偏差分析
```

## 合规运营

### 月度合规检查（法务合规员）

```
1. 监管动态
   - 影响产品的新法规
   - 现有法规变更
   - 行业执法行动
   - 合规截止日期追踪

2. 隐私合规
   - 数据主体请求处理
   - 知情同意管理有效性
   - 数据留存策略执行
   - 跨境传输合规

3. 安全合规
   - 漏洞扫描结果
   - 补丁管理状态
   - 访问控制复查
   - 事件日志复查

4. 审计就绪度
   - 文档是否最新
   - 证据收集状态
   - 培训完成率
   - 政策确认追踪
```

## 策略演进

### 季度策略复盘（工作室制片人）

```
1. 市场定位评估
   - 竞争格局变化（趋势研究员输入）
   - 市场份额变化
   - 品牌认知（品牌守护者输入）
   - 客户满意度趋势（反馈分析师输入）

2. 产品策略
   - 功能路线图复盘
   - 技术债评估（工具评估师输入）
   - 平台扩展机会
   - 合作伙伴评估

3. 增长策略
   - 渠道有效性复盘
   - 新市场机会
   - 定价策略评估
   - 扩张规划

4. 组织健康
   - 流程效率（工作流优化师输入）
   - 团队绩效指标
   - 资源配置优化
   - 能力建设需求

产出：季度策略评估 → 更新路线图和优先级
```
