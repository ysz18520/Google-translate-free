# 第 0 阶段手册 — 情报与发现

> **时长**：3-7 天 | **智能体**：6 个 | **守门人**：高管摘要生成器

---

## 目标

在投入资源之前先验证机会。问题、市场和监管环境没搞清楚之前不动手。

## 前提条件

- [ ] 有项目简报或初始概念
- [ ] 确定了利益相关方赞助人
- [ ] 发现阶段的预算已批准

## 智能体激活顺序

### 第一波：并行启动（第 1 天）

#### 趋势研究员 — 市场情报负责人
```
激活趋势研究员，做 [项目领域] 的市场情报收集。

交付要求：
1. 竞争格局分析（直接 + 间接竞争对手）
2. 市场规模估算：TAM、SAM、SOM 及方法论
3. 趋势生命周期图谱：这个市场在采用曲线的什么位置？
4. 3-6 个月趋势预测及置信区间
5. 该领域的投融资趋势

来源：至少 15 个独立的、经过验证的来源
格式：策略报告 + 高管摘要
时间线：3 天
```

#### 反馈分析师 — 用户需求分析
```
激活反馈分析师，做 [项目领域] 的用户需求分析。

交付要求：
1. 多渠道反馈收集计划（问卷、访谈、评论、社交媒体）
2. 现有用户触点的情感分析
3. 痛点识别和排序（RICE 评分）
4. 功能需求分析及业务价值估算
5. 反馈模式中的流失风险指标

格式：综合反馈报告 + 优先级矩阵
时间线：3 天
```

#### UX 研究员 — 用户行为分析
```
激活 UX 研究员，做 [项目领域] 的用户行为分析。

交付要求：
1. 用户访谈计划（5-10 个目标用户）
2. 用户画像构建（3-5 个主要画像）
3. 主要用户流程的旅程图
4. 竞品的可用性启发式评估
5. 有统计验证的行为洞察

格式：调研发现报告 + 用户画像和旅程图
时间线：5 天
```

### 第二波：并行启动（第 1 天，与第一波独立）

#### 数据分析师 — 数据基础评估
```
激活数据分析师，做 [项目领域] 的数据基础评估。

交付要求：
1. 现有数据源审计（有哪些可用数据？）
2. 信号识别（我们能测量什么？）
3. 基线指标建立
4. 数据质量评估及完整性评分
5. 分析基础设施建议

格式：数据审计报告 + 信号地图
时间线：2 天
```

#### 法务合规员 — 监管扫描
```
激活法务合规员，做 [项目领域] 的监管扫描。

交付要求：
1. 适用的监管框架（GDPR、CCPA、HIPAA 等）
2. 数据处理要求和约束
3. 目标市场的司法管辖区映射
4. 合规风险评估及严重度评级
5. 阻塞性 vs 可管理的合规问题

格式：合规要求矩阵
时间线：3 天
```

#### 工具评估师 — 技术版图
```
激活工具评估师，做 [项目领域] 的技术版图评估。

交付要求：
1. 该问题域的技术栈评估
2. 关键组件的自建 vs 采购分析
3. 与现有系统的集成可行性
4. 开源 vs 商业方案评估
5. 技术风险评估

格式：技术栈评估 + 推荐矩阵
时间线：2 天
```

## 汇聚点（第 5-7 天）

六个智能体全部交付报告。高管摘要生成器进行综合：

```
激活高管摘要生成器，综合第 0 阶段的发现。

输入文档：
1. 趋势研究员 → 市场分析报告
2. 反馈分析师 → 综合反馈报告
3. UX 研究员 → 调研发现报告
4. 数据分析师 → 数据审计报告
5. 法务合规员 → 合规要求矩阵
6. 工具评估师 → 技术栈评估

输出：高管摘要（不超过 500 字，SCQA 格式）
需要的决策：继续 / 终止 / 调整方向
包含：量化的市场机会、验证过的用户需求、监管路径、技术可行性
```

## 质量门禁检查清单

| # | 标准 | 证据来源 | 状态 |
|---|------|---------|------|
| 1 | 市场机会验证通过，TAM 超过最低可行阈值 | 趋势研究员报告 | |
| 2 | 至少 3 个有数据支撑的已验证用户痛点 | 反馈分析师 + UX 研究员 | |
| 3 | 没有阻塞性合规问题 | 法务合规员矩阵 | |
| 4 | 关键指标和数据源已确定 | 数据分析师审计 | |
| 5 | 技术栈可行且已评估 | 工具评估师评估 | |
| 6 | 高管摘要已交付，含继续/终止建议 | 高管摘要生成器 | |

## 门禁决策

- **继续**：进入第 1 阶段 — 策略与架构
- **终止**：归档发现，记录教训，资源转向
- **调整方向**：根据发现修改范围/方向，针对性地重新跑发现

## 交接给第 1 阶段

```markdown
## 第 0 阶段 → 第 1 阶段交接包

### 带入的文档：
1. 市场分析报告（趋势研究员）
2. 综合反馈报告（反馈分析师）
3. 用户画像和旅程图（UX 研究员）
4. 数据审计报告（数据分析师）
5. 合规要求矩阵（法务合规员）
6. 技术栈评估（工具评估师）
7. 含"继续"决策的高管摘要（高管摘要生成器）

### 识别到的关键约束：
- [法务合规员发现的监管约束]
- [工具评估师发现的技术约束]
- [趋势研究员发现的市场时机约束]

### 优先用户需求（给 Sprint 排序师）：
1. [痛点 1 — 来自反馈分析师]
2. [痛点 2 — 来自 UX 研究员]
3. [痛点 3 — 来自反馈分析师]
```

---

*当高管摘要生成器根据六个发现智能体的证据给出"继续"决策时，第 0 阶段结束。*
