## 问题记录

### 问题 1：删除产品/页面后翻译数据残留
**状态：✅ 已解决（v1.2.1）**
**方案：B + D 组合**
- B（Webhook 实时清理）：监听 `products/delete`、`pages/delete`，删除资源时同步删除翻译记录
- D（全站翻译前兜底）：每次「全站翻译」前扫描数据库，清理已不存在资源的翻译

### 问题 2：翻译范围过大（草稿/未发布也被翻译）
**状态：✅ 已解决（v1.2.1）**
- `getAllProducts()` 过滤：`status === 'active'`
- `getAllPages()` 过滤：`published_at !== null`
- 仅翻译顾客可见的内容，节省 API 调用

## 迭代历程（功能更新与架构演进）

### v1.0.6（2026-05-22）— 初始稳定基线
**核心功能上线**
- OAuth 2.0 授权流程（Shopify 官方 SDK）
- 前端悬浮语言切换器（右下角/左下角）
- 动态文本替换（切换语言时不保留原文）
- 批量翻译（商品 + 页面 + Theme）
- 翻译缓存系统（`translated` / `outdated` / `pending`）
- Webhook 自动更新检测（`products/update`、`pages/update`、`themes/update`）
- RTL 支持（阿拉伯语从右到左布局）
- 店主调试工具（`Ctrl + Shift + L`）
- Docker + Nginx 部署

### v1.1.0（2026-05-25）— 语言配置完整版
**语言系统扩展**
- 支持默认语言配置
- 支持自动检测浏览器语言
- 支持自定义语言添加（添加后自动全量翻译）
- 5 种必备语言（en/es/fr/ara/zh）强制启用

### v1.2.0（2026-05-26）— 源语言自动检测 + 架构优化
**重大架构调整**
- **源语言自动检测**：从网站 `<html lang>` 读取，不再硬编码 'en'
- **移除自动翻译**：首次访问始终显示源语言，用户手动切换才翻译
- **简化语言配置**：移除默认语言下拉框、移除浏览器语言检测开关
- **AI Prompt 优化**：强制翻译所有内容（含产品名、品牌名）
- **widget.js v5.3 架构**：WeakMap 文本节点替换，无 innerHTML 重建、无脚本重执行
- **bfcache 兼容**：页面从浏览器缓存恢复后自动重新应用翻译
- **实时翻译兜底**：未命中缓存时批量调百度 API，结果回写数据库
- **加载动画优化**：橙色脉冲 + spinner，无全屏白屏遮挡

### v1.2.1（2026-05-26）— 翻译清理 + 范围过滤
**数据质量优化**
- 暂停按钮与全部翻译状态同步修复
- 仅翻译 `active` 状态产品和 `published_at !== null` 的页面
- Webhook 实时清理（`products/delete`、`pages/delete`）
- 全站翻译前兜底扫描，清理无效翻译记录
