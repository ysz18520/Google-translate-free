# AutoTranslate Pro - 安装与使用指南

> 版本：v1.2.1
> 更新日期：2026-05-26

## 一、安装前准备

### 1.1 需要向店铺 Owner 收集的信息

| 信息项 | 说明 | 示例 |
|--------|------|------|
| **Shopify 店铺域名** | 格式为 `xxx.myshopify.com` | `test-store.myshopify.com` |
| **店铺后台登录邮箱** | 用于确认 Owner 身份 | `owner@example.com` |
| **网站源语言** | 店铺默认显示的语言（从 `<html lang>` 自动检测） | English、中文等 |

### 1.2 你需要准备的信息

| 信息项 | 说明 | 填写位置 |
|--------|------|----------|
| **Shopify App 凭证** | API Key、App Secret | `.env` 文件 |
| **翻译 API 密钥** | 百度翻译 App ID + Secret Key | `.env` 文件 |
| **应用域名** | 你的服务器域名，如 `admin.coollaa.com` | `.env` 文件 + Shopify Partners |
| **数据库连接** | MySQL 连接字符串 | `.env` 文件 |

## 二、安装步骤

> **重要前提**：你的应用必须先在 Shopify Partners 后台配置好以下内容，否则授权会报错"此应用尚无法安装"。
>
> 1. **App URL**：`https://admin.coollaa.com`
> 2. **Allowed redirection URL(s)**：`https://admin.coollaa.com/api/shopify/callback`
> 3. **分发方式**：自定义分发（见下方步骤）

---

### 方式一：自定义分发链接安装（推荐）

这是给普通店铺安装的标准流程。**每次给新店铺安装，都需要在 Partners 后台生成一次链接。**

#### Step 1：获取店铺域名

向店铺 Owner 索要 Shopify 店铺域名，格式为 `xxx.myshopify.com`：

```
215c1a-3.myshopify.com
test-store.myshopify.com
```

#### Step 2：在 Partners 后台生成分发链接

1. 登录 [Shopify Partners](https://partners.shopify.com)
2. 进入 **Apps** → **AutoTranslate Pro** → **Configuration** → **Distribution（分发）**
3. 选择 **"自定义分发"**
4. **商店域名** 填入目标店铺域名，如 `215c1a-3.myshopify.com`
5. **取消勾选** "允许同一 Plus 组织进行多商店安装"（普通店铺不需要）
6. 点击 **"生成链接"**

#### Step 3：发送安装链接给店铺 Owner

生成后，Shopify 会给你一个自定义安装链接，格式类似：

```
https://partners.shopify.com/xxx/apps/xxx/installations/xxx
```

通过微信/邮件/钉钉等方式发送给店铺 Owner。

#### Step 4：Owner 完成授权

店铺 Owner 点击链接后的操作流程：

1. **登录 Shopify 账号**
   - 如果未登录，会跳转到 Shopify 登录页
   - 使用店铺 Owner 的账号密码登录

2. **查看应用权限**
   - 应用会请求以下权限：
     - `read_products` — 读取商品信息
     - `read_content` — 读取页面内容
     - `read_themes` — 读取主题文件
     - `read_script_tags` — 读取已注入的脚本
     - `write_script_tags` — 注入翻译脚本到店铺
   - 点击 **"安装应用"**

3. **授权完成**
   - 自动跳转到管理后台
   - 同时前端脚本已自动注入到店铺主题中

#### Step 5：确认安装成功

在店铺前台页面（非后台）刷新查看：
- 右下角应出现 **地球图标悬浮按钮**
- 点击后弹出语言选择下拉菜单
- 如果没有出现，参考下方「常见问题」排查

---

### 方式二：从 Shopify Partners 后台直接安装（开发测试用）

仅用于你拥有的开发店铺测试：

1. 登录 [Shopify Partners](https://partners.shopify.com)
2. 进入你的应用管理页
3. 点击 **"Select store"** → 选择目标开发店铺
4. 点击 **"Install app"**
5. 安装完成后访问 `https://admin.coollaa.com/translator?shop=店铺域名.myshopify.com`

## 三、首次配置（管理后台）

授权成功后，进入管理后台进行首次配置。

### 3.1 访问管理后台

```
https://admin.coollaa.com/translator?shop=你的店铺.myshopify.com
```

### 3.2 概览页确认

概览页显示以下信息：
- 已授权店铺名称
- 店铺域名
- 翻译统计（已翻译 / 未翻译数量）
- 支持的语言列表

### 3.3 设置页配置

切换到 **"设置"** 标签：

| 配置项 | 说明 | 推荐值 |
|--------|------|--------|
| **悬浮按钮位置** | 语言切换器显示位置 | 右下角 |
| **主题色** | 悬浮按钮颜色 | `#1976d2`（蓝色） |
| **启用语言** | 店铺支持的语言 | 默认 5 种全选 |
| **网站原文语言** | 自动检测，不可修改 | 只读显示 |

**注意：**
- 5 种必备语言（英/西/法/阿/中）**强制启用**，不可取消
- 网站原文语言从店铺 `<html lang>` 属性自动读取，无需手动配置

### 3.4 首次全站翻译

**必须先完成翻译，前台切换语言才能生效。**

#### 步骤 1：翻译 Theme（主题 UI 文本）

1. 在设置页找到 **"翻译 Theme"** 按钮
2. 点击后开始翻译主题的 locales 文件和导航菜单
3. 等待进度完成（约 1-3 分钟，取决于主题大小）
4. 完成后按钮显示 **"已翻译"**

#### 步骤 2：全站翻译（商品 + 页面）

1. 切换到 **"首页"** 标签
2. 点击 **"全站翻译"** 按钮
3. 系统会自动：
   - 获取所有商品标题和描述
   - 获取所有页面内容
   - 调用翻译 API 逐条翻译
   - 将结果写入数据库缓存
4. 等待翻译完成（时间取决于商品数量，通常 5-15 分钟）

#### 步骤 3：验证翻译效果

1. 打开店铺前台首页
2. 点击右下角地球图标
3. 选择 **"Español"** 或 **"中文"**
4. 页面文字应在 1-3 秒内完成翻译

## 四、日常使用

### 4.1 店铺前台（顾客视角）

#### 切换语言

1. 点击右下角 **地球图标** 悬浮按钮
2. 在弹出菜单中选择目标语言：
   - 🇺🇸 English（英语）
   - 🇪🇸 Español（西班牙语）
   - 🇫🇷 Français（法语）
   - 🇸🇦 العربية（阿拉伯语）
   - 🇨🇳 中文
3. 页面文字自动替换为目标语言

#### 首次访问行为

- **外部进入**（搜索引擎、直接访问、书签）：默认显示**网站原文语言**，不自动翻译
- **同域跳转/刷新**：如果之前手动切换过语言，会保留上次的选择
- **切换后翻译**：首次切换需要 1-3 秒（调用翻译 API），第二次切换秒开（缓存命中）

#### RTL 支持

切换阿拉伯语（العربية）时，页面自动调整为**从右到左（RTL）**布局。

### 4.2 管理后台（店主视角）

#### 查看翻译统计

在首页查看：
- 已翻译商品数 / 总商品数
- 已翻译页面数 / 总页面数
- Theme 翻译状态

#### 添加自定义语言

1. 进入 **"设置"** 页
2. 点击 **"添加自定义语言"**
3. 填写：
   - 语言代码（如 `de` 表示德语）
   - 语言名称（如 `Deutsch`）
   - 国旗 emoji（如 `🇩🇪`）
4. 点击确认后，系统自动启动全站翻译（新语言）

#### 更新翻译（内容变更后）

当修改了商品/页面/主题后，Shopify 会自动通知插件更新。如需立即更新：

**方式一：管理后台刷新**
1. 进入首页
2. 点击 **"全站翻译"** 重新翻译所有内容

**方式二：调试模式刷新**
1. 在首页点击 **"进入店铺调试模式"**
2. 生成调试链接，点击跳转到店铺前台
3. 按 `Ctrl + Shift + L` 弹出调试菜单
4. 选择 **"刷新当前页翻译"** 或 **"刷新全站翻译"**

## 五、多店铺管理

### 5.1 添加新店铺

**普通店铺（非 Plus）：每次都需要重新生成分发链接。**

流程：
1. 获取新店铺的 `xxx.myshopify.com` 域名
2. 登录 Partners 后台 → 你的应用 → Distribution
3. 修改自定义分发的目标店铺域名为新店铺
4. 重新生成链接
5. 发送给新店铺的 Owner

**Shopify Plus 组织店铺：**

如果多个店铺属于同一 Plus 组织，勾选了"允许同一 Plus 组织进行多商店安装"后，**同一个链接可以给该组织下的所有店铺使用**。

### 5.2 切换管理不同店铺

访问管理后台时带上 `shop` 参数：

```
https://admin.coollaa.com/translator?shop=店铺A.myshopify.com
https://admin.coollaa.com/translator?shop=店铺B.myshopify.com
```

### 5.3 数据隔离

| 项目 | 说明 |
|------|------|
| 翻译数据 | 每个店铺独立存储，互不干扰 |
| 设置配置 | 每个店铺独立的 widget 位置和颜色 |
| API 调用 | 所有店铺共用同一套翻译 API 额度 |
| 数据库记录 | 按 `shopId` 隔离 |

## 六、常见问题排查

### 6.1 店铺前台看不到语言切换器

| 排查步骤 | 操作 |
|----------|------|
| 1. 确认应用已安装 | Shopify 后台 → 应用 → 查看是否有 AutoTranslate Pro |
| 2. 确认已完成授权 | 重新访问授权链接，看是否提示已安装 |
| 3. 检查浏览器控制台 | F12 → Console，搜索 "Coollaa"，看是否有初始化日志 |
| 4. 检查网络请求 | F12 → Network，搜索 `translator-widget.js`，看是否 200 |
| 5. 强制刷新页面 | `Ctrl + Shift + R` 或无痕模式打开 |

### 6.2 切换语言后页面没变化

| 排查步骤 | 操作 |
|----------|------|
| 1. 确认已完成首次翻译 | 管理后台 → 首页 → 查看翻译统计是否大于 0 |
| 2. 检查浏览器控制台 | F12 → Console，看是否有红色报错 |
| 3. 确认不是源语言 | 源语言切换不会触发翻译（自动检测的默认语言） |
| 4. 等待翻译完成 | 首次切换需要 1-3 秒，按钮会显示橙色加载动画 |
| 5. 刷新缓存 | 管理后台 → 调试模式 → 刷新全站翻译 |

### 6.3 翻译内容显示异常（乱码/混合语言）

| 排查步骤 | 操作 |
|----------|------|
| 1. 强制刷新页面 | `Ctrl + Shift + R` |
| 2. 清除浏览器缓存 | 无痕模式打开验证 |
| 3. 重新翻译 | 管理后台 → 全站翻译 |
| 4. 检查数据库编码 | 确保 MySQL 使用 `utf8mb4` 字符集 |

### 6.4 授权失败或回调报错

| 排查步骤 | 操作 |
|----------|------|
| 1. 检查 `.env` 配置 | 确认 `SHOPIFY_API_KEY`、`SHOPIFY_APP_SECRET` 正确 |
| 2. 检查 Redirect URI | Shopify Partners 后台配置的回调地址必须与服务器的 `SHOPIFY_REDIRECT_URI` 一致 |
| 3. 检查应用域名 | 确保 `SHOPIFY_APP_URL` 与实际域名一致 |
| 4. 查看服务器日志 | `docker logs shanzhao-translator` |

## 七、卸载应用

### 7.1 店铺 Owner 自行卸载

1. 进入 Shopify 后台 → 应用
2. 找到 AutoTranslate Pro
3. 点击 **"删除"** 或 **"卸载"**
4. 系统会自动：
   - 移除注入的 `translator-widget.js` ScriptTag
   - 移除注册的 Webhook
   - 保留数据库记录（如需彻底删除请联系管理员）

### 7.2 管理员强制移除

如需从服务端移除店铺数据：

```bash
# 进入服务器
cd /shanzhao/translator

# 手动删除 ScriptTag（如需）
# 手动删除数据库记录（如需）
```
